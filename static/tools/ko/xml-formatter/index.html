📄 XML Formatter

XML을 보기 좋게 포맷하거나 압축합니다. 유효성도 즉시 검사합니다.

❓ 자주 묻는 질문

XML(Extensible Markup Language)은 데이터를 구조화하여 저장하고 전송하기 위한 마크업 언어입니다. HTML과 비슷하지만 사용자 정의 태그를 사용할 수 있습니다. RSS, SOAP, SVG, Office 문서 형식 등에 널리 사용됩니다.
XML은 태그 기반으로 속성(attribute)과 네임스페이스를 지원하며 문서 중심 데이터에 강합니다. JSON은 더 간결하고 JavaScript와의 통합이 자연스러우며 현대 API에서 선호됩니다. 복잡한 문서 구조나 메타데이터가 많은 경우 XML이 유리합니다.
XML이 올바른 형식(well-formed)인지 확인합니다. 모든 태그는 닫혀야 하고, 속성값은 따옴표로 감싸야 하며, 특수문자는 이스케이프해야 합니다. DTD나 XML Schema를 통한 구조 검증은 별도의 도구가 필요합니다.
XML에서는 &, <, >, ', " 다섯 가지 엔티티를 사용해 특수문자를 이스케이프합니다. 예를 들어 <는 &lt;, &는 &amp;로 표기합니다.
CDATA(Character Data) 섹션은 XML 파서가 해석하지 않는 텍스트 블록입니다. <![CDATA[...]]> 형식으로 표기하며, HTML이나 스크립트 코드처럼 특수문자가 많은 내용을 그대로 포함할 때 사용합니다.
