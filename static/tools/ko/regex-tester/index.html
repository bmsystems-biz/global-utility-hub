🔍 Regex 테스터

정규식을 실시간으로 테스트하고 매치 결과를 확인합니다.

플래그:
테스트 문자열과 패턴을 입력하면 매치 결과가 표시됩니다.

❓ 자주 묻는 질문

정규식은 문자열 패턴을 정의하는 언어입니다. 이메일 주소, 전화번호, URL 등의 패턴을 검색하고 추출하거나 검증하는 데 사용됩니다. 프로그래밍 언어 대부분에서 지원합니다.
\d (숫자), \w (단어문자), \s (공백), . (임의문자), ^ (줄 시작), $ (줄 끝), + (1회이상), * (0회이상), ? (0또는1회), {n,m} (n~m회), [abc] (문자집합), (abc) (그룹 캡처)
간단한 이메일 패턴: ^[\w.-]+@[\w.-]+\.[a-zA-Z]{2,}$ — 실무에서는 이메일 형식이 매우 복잡하므로 완벽한 정규식보다 서버에서 이메일 발송 확인 방식을 권장합니다.
(?=...) 은 긍정 전방탐색(다음에 패턴이 있을 때), (?!...) 은 부정 전방탐색, (?<=...) 은 긍정 후방탐색(이전에 패턴이 있을 때), (?<!...) 은 부정 후방탐색입니다. 실제로 매치는 하지 않고 조건만 확인합니다.
한국 휴대폰: ^01[016789]-?\d{3,4}-?\d{4}$ / 일반전화: ^0\d{1,2}-?\d{3,4}-?\d{4}$ / 통합: ^(01[016789]|0[2-9]\d?)-?\d{3,4}-?\d{4}$
