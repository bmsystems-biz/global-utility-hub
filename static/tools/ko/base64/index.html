🔤 Base64 인코더/디코더

텍스트를 Base64로 인코딩하거나 디코딩합니다. 한글, 이모지 등 유니코드를 지원합니다.

❓ 자주 묻는 질문

Base64는 이진 데이터를 ASCII 텍스트로 변환하는 인코딩 방식입니다. A-Z, a-z, 0-9, +, / 64개의 문자를 사용합니다. 이메일 첨부파일, 이미지의 data URI, JWT 토큰 등에 널리 사용됩니다.
아닙니다. Base64는 암호화가 아니라 인코딩입니다. 누구나 디코딩할 수 있으며 보안을 제공하지 않습니다. 비밀 정보를 숨기려면 AES, RSA 같은 실제 암호화를 사용해야 합니다.
일반 Base64의 + 와 / 문자는 URL에서 특별한 의미를 가져 문제가 발생할 수 있습니다. URL-safe Base64는 +를 -, /를 _로 대체하여 URL 파라미터에 안전하게 사용할 수 있도록 합니다. JWT 토큰에 사용됩니다.
Base64는 원본 데이터보다 약 33% 크기가 증가합니다. 3바이트씩 묶어 4개의 ASCII 문자로 변환하기 때문입니다. 예를 들어 300KB 파일은 Base64 인코딩 후 약 400KB가 됩니다.
가능합니다. <img src="data:image/png;base64,인코딩값"> 형식으로 이미지를 직접 HTML에 포함할 수 있습니다. 작은 아이콘에 유용하지만, 큰 이미지는 파일 크기가 커져 성능에 좋지 않습니다.
