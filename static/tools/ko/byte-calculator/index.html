인코딩별 바이트 설명
• 영문/숫자/특수문자: UTF-8 1B / UTF-16 2B / EUC-KR 1B
• 한글: UTF-8 3B / UTF-16 2B / EUC-KR 2B
• 이모지: UTF-8 4B / UTF-16 4B

📐 바이트 계산 공식

UTF-8 바이트 계산: • ASCII(영문, 숫자) → 1 byte/char • 한글, 한자, 일본어 → 3 bytes/char • 이모지 → 4 bytes/char EUC-KR 계산: • ASCII → 1 byte/char • 한글 → 2 bytes/char

❓ 자주 묻는 질문

UTF-8은 전 세계 모든 문자를 표현하는 국제 표준이고, EUC-KR은 한국어에 특화된 인코딩입니다. 한글의 경우 UTF-8은 3바이트, EUC-KR은 2바이트를 사용합니다. 현재 대부분의 웹사이트는 UTF-8을 사용합니다.
MySQL 등의 데이터베이스에서 VARCHAR 컬럼은 바이트 기준으로 크기를 제한합니다. 한글이 포함된 경우 UTF-8 기준으로 1글자당 최대 3바이트가 필요하므로, 글자 수의 3배 이상으로 컬럼 크기를 설정해야 합니다.
RFC 표준에 따르면 이메일 한 줄은 998바이트 이하여야 합니다. 대부분의 이메일 서비스는 전체 본문 크기를 수 MB 이하로 제한합니다.
기본 SMS는 GSM-7 인코딩 기준 160자(160바이트)입니다. 한글이 포함되면 유니코드(UCS-2) 방식으로 전환되어 70자 제한이 됩니다.
1KB = 1,024바이트, 1MB = 1,024KB = 1,048,576바이트입니다. SI 단위(kB)에서는 1kB = 1,000바이트로 표기하기도 합니다.
